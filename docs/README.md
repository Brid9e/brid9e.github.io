开发日志放在本目录，构建时打包进站点。

- **目录**：可按年份分子目录，例如 **`docs/2026/site-notes.md`**
- **文件名**：**不必**再带 `YYYY-MM-DD-` 前缀；每篇须在 **frontmatter** 里写 **`date`**（见下）。仍兼容旧约定 **`YYYY-MM-DD-短名.md`**：若文件名以日期开头，在未写 frontmatter `date` 时可用文件名里的日期作默认
- **正文**：可用第一行 `# 标题`；也可用 frontmatter 的 **`title`** 单独指定标题（正文可不再写一级标题）
- **摘要**：构建时仍会生成 `excerpt` 字段（供以后扩展），当前首页与列表均**不展示摘录**

### 日期与时间（写在 frontmatter）

与 VitePress 类似，在文件**最开头**加 YAML 块，用 `date` 指定发布日或时间：

- **只写日期**：

```yaml
---
date: 2026-03-18
title: 可选覆盖标题
---
```

- **要写具体时刻**（时区、UTC、本地时间均可，按字符串能 `Date` 解析为准）：

```yaml
---
date: 2026-03-18T15:30:00+08:00
---
```

或 `date: "2026-03-18 15:30"` 等形式。带时间的条目在列表里会显示为 `M/DD HH:mm`，详情页显示完整日期时间。

若文件名是 **`YYYY-MM-DD-*.md`** 且**未**写 frontmatter `date`，则使用**文件名**里的 `YYYY-MM-DD`（当天 12:00 本地时间参与排序）。
